# AGI 指揮中心：智慧調度標準規範書 (Orchestration SOP)

## 1. 願景
本專案不只是一個管理後台，而是一個 **AI 指揮的工作站**。其核心競爭力在於「AI 自動調度工具」，而非「使用者手動操作軟體」。

## 2. 系統雙區規劃
### A. AGI 協作中心 (Neural Brain)
- **地位**: 系統第一優先入口。
- **功能**: 透過 AGI 理解自然語言，自動從「工具房」提取組件至對話中。
- **守則**: 保持極度乾淨，只在必要時召喚積木。

### B. 功能工具房 (Functional Toolkit)
- **地位**: 系統的基礎設施。
- **功能**: 提供全量數據管理、批次設定與專業巡檢。
- **守則**: 提供高擴充性的積木，並為每個積木預留 AGI 調度接口。

## 3. 積木開發標準 (Block Integration Standard)
開發新功能（工具）時，必須包含以下三個維度：
1. **Full Page**: 獨立的完整管理頁面（存放在側欄選單）。
2. **Micro UI**: 專供 AGI 召喚到對話窗的小型 UI 組件。
3. **Intent Link**: 定義 AGI 在何種語境下應觸發此積木。

## 4. 隔離與安全 (Isolation)
- 每個工具必須嚴格遵守 `oa_id` 數據隔離。
- AGI 跨店取用工具時，必須進行顯性告知與權限校驗。

---
*「讓 AGI 指揮積木，讓使用者指揮 AGI。」—— 本專案核心設計準則。*
